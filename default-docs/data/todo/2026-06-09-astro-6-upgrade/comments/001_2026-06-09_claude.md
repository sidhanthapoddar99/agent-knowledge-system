# Kickoff — why now

Current pin: `astro@5.16.12` (`astro-doc-code/package.json`). Latest: **6.4.5** — a full major behind.

The mechanical bump is trivial; the reason this is an *issue* and not a one-line PR is the surface area this framework occupies inside Astro's internals:

- Custom **dev-toolbar integration** (`src/dev-tools/integration.ts`) and middleware — integration API is a known break-point across Astro majors.
- The **SSR module-invalidation** cache work (`moduleGraph.invalidateModule`, dual-invalidation for Vite SSR module isolation) — recent, hard-won, and tightly coupled to the Vite version Astro vendors.
- A bundled **Vite alias set** + `fs.allow` config in `astro.config.mjs`.

So the plan is: document the 6.x changes and the per-subsystem impact (the two `notes/` groups) before touching `package.json`. Subtasks land once we've read the migration guide against our actual code.

The dependency-reinstall question about `./start` is answered next, in `comments/002`.
