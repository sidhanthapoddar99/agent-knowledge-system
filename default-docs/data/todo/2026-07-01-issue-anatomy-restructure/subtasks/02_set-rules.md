Decide and write down the operating rules for the new structure, so the sections
don't blur back together in practice.

- [ ] When to create a new issue vs add to an existing one
- [ ] When something is Brainstorm vs Notes vs Subtask vs Agent-log
- [ ] When deliberation graduates from `brainstorm/` into `notes/`
- [ ] Rules for Comments staying lean (evolution log, not a forum)
- [ ] **A dedicated "issues dump"** — a lightweight place to jot down random ideas /
      half-formed issues without the ceremony of a full issue folder. Decide its
      shape (a single backlog issue? a top-level `inbox/`? captured via a command —
      see subtask 04) and the rule for graduating a dump entry into a real issue.
