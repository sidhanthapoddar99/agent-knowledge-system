How artifacts get their colors

Every .html artifact in this framework travels the same pipeline and picks one of two theme modes. This page diagrams that system — and is itself a live theme: "site" example: its backgrounds, text, borders and spacing all consume the injected host theme; only the diagram strokes below (flow arrows, branch markers) are local elemental color.

2 · Two theme modes

What the sidecar's artifact.theme decides

Every sidecar declares one of two modes. The route reads it once, per request, and branches before a byte is served.

sidecar artifact.theme
theme: site
  • Route injects the site's resolved theme CSS at the start of <head>, via getThemeCSS(getTheme()), plus a dark-mode init script.
  • Route stamps data-theme; the artifact consumes var(--color-*) tokens directly — no local palette.
  • Re-themes live: scripts/artifacts.ts watches the parent's data-theme attribute and mirrors it onto the embed's iframe.contentDocument on every toggle.
theme: self (default)
  • Route serves the file byte-for-byte, untouched — nothing injected.
  • The artifact carries its own dual-theme system: :root, @media (prefers-color-scheme: dark), and :root[data-theme] overrides.
  • The site toggle still wins: the embed still receives the mirrored data-theme attribute, so the artifact's own [data-theme] selectors respond.

3 · Where colors come from

A decision by the artifact's subject

The mode choice isn't stylistic — it follows what the artifact is: a page that shows data inherits the host; a page whose subject is a look owns its own.

What is the artifact's subject?
explanatory / data → site

Consumes the injected vocabulary, each with a neutral fallback:

--color-bg-primary --color-text-primary --color-border-default --spacing-md --ui-text-body
needs a color the contract lacks? Diagram strokes, node fills, series hues — add them locally in the HTML. Ambient (bg / text / border / spacing) stays injected. This page's own --el-flow / --el-site / --el-self / --el-neutral are exactly that exception.
design / UI-UX → self

Owns its values on :root + dark overrides. Contract names preferred over invented ones:

--color-bg-primary --color-text-primary --color-brand-primary

A new name only when the role goes beyond the contract's scope — e.g. --color-accent-soft.
