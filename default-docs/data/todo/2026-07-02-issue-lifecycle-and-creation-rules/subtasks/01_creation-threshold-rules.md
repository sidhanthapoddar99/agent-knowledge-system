Write the operating rules for *when the ceremony of a full issue folder is earned*:

- [ ] Full issue — coherent thinking+execution unit; must sustain ≥1 real subtask
- [ ] Subtask — one-prompt fixes and small features attach to an existing issue's
      center of gravity instead of getting a folder
- [ ] Brainstorm entry — deliberation/research toward a decision lives inside the
      issue it informs; standalone deliberation-issues are an anti-pattern
- [ ] Fold-in / supersession rule (distilled in
      `2026-07-01-issue-anatomy-restructure/subtasks/05`): shipped-work issues stay
      and close with a supersession comment; pure-deliberation issues fold into the
      winner's `brainstorm/` and are deleted (git keeps history)
- [ ] Decide where the rules live (user-guide `19_issues` page + skill reference)

Related: `2026-07-01-issue-anatomy-restructure/subtasks/02_set-rules.md` — owns the
anatomy-side boundaries (brainstorm vs notes vs agent-log, comments hygiene, issues
dump). This subtask owns the *threshold* question; keep the two linked, not duplicated.
