Artifacts as a first-class content type

Self-contained HTML pages embed in the docs content area and open full-page at a reserved /artifacts/<path> route — with the URL as the primitive the embed is built on, not an afterthought.

status open priority high component components label feature

The vision

One artifact, two surfaces — the URL underneath both

An NN_-prefixed .html file appears in the sidebar like any page. It renders two ways, and the second is the foundation of the first.

Embedded

In the content area

Selecting the artifact replaces the central content column with the embedded page (an iframe). The navigational sidebar stays; the outline column, having no headings to list, collapses and hands its width over.

Full page

Its own reserved route

An "open full page" affordance navigates to a route where the artifact is the page — full viewport, bookmarkable, shareable.

/artifacts/<path-to-artifact>

URL as the primitive. The embed's iframe needs a src regardless, and that src is the route — so the route is the dependency the embed is built on, not an alternative to it. It also sidesteps the Fullscreen API's weaknesses: a real URL is linkable, escapable, and durable.

Settled decisions

Six choices pinned for the implementer

The finalized record a downstream implementer cites — each fixed on 2026-07-07, each with its rationale attached so it survives a cold read.

01

The dedicated URL is the primitive

/artifacts/<path> is the primary full-view mechanism; the in-content embed is built on top of it because the iframe's src is that route.

02

Embed affordances

"Open full page" is the primary control — a plain <a href> to the route. An in-place expand may exist as a secondary, optional affordance.

03

Metadata in a .meta.json sidecar

Same-name sidecar (never injected frontmatter) keeps the HTML a pristine standalone document — and carries explicit declared values so an agent skips parsing the markup.

04

Reserved-URL set, guarded at config load

No section may use artifacts as a base URL — nor assets, content-assets, api, editor. A hard throw, not a warning.

05

First-party trust boundary

Artifacts serve same-origin and run unsandboxed as first-party content. text/html is scoped to the route, deliberately kept out of the shared MIME map.

06

"Chrome stays" means sidebar-only

An artifact has no headings, so the TOC column would be dead chrome. The sidebar stays; the outline collapses and its width goes to the artifact.
