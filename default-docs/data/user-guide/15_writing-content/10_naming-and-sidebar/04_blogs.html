Blogs — File Tree → Sidebar

Blog is the simplest naming scheme in the framework: a flat folder of YYYY-MM-DD-<slug>.md files. The date drives ordering (newest-first) and is stripped from the URL; there is no NN_ prefix and no sidebar tree — the index renders as a grid of cards.

On disk → the rendered index

Files are named by date, and can sit in the folder in any order. The loader sorts by date descending, so the newest post is first regardless of on-disk order. Each card links to /blog/<slug> — the date prefix is removed from the URL.

The card shows title (frontmatter), description, up to 2 tags, optional author, and the formatted date (en-US, e.g. "Mar 2, 2024"). A featured image renders above the card when present.

Edge cases

Where the date comes from, what happens when it is wrong or absent, drafts, and why a numeric prefix means nothing here.

Frontmatter date wins

2024-01-15-post.md
  ↳ date: 2024-06-01

The filename date is only a fallback: it is applied only when frontmatter has no date. A frontmatter date overrides it, so this post sorts and displays as Jun 1, 2024 — not Jan 15.

No date anywhere → sorts last

notes.md
  (no date prefix, no fm date)

Filename without a YYYY-MM-DD- prefix is allowed: the whole name becomes the slug and date stays undefined. In the sort it counts as time 0, so it falls to the oldest end (last, newest-first). The card renders no date line.

Invalid date is not caught. A malformed date like 2024-13-40 (in the filename or frontmatter) is never validated. new Date() yields Invalid Date → getTime() is NaN, so the sort comparison is NaN and that post's position becomes unstable; the card prints the literal text "Invalid Date". No build error, no warning — it silently misbehaves.

draft: true

2024-05-01-wip.md
  ↳ draft: true

Visible in dev, hidden in production. Drafts are included when includeDrafts is on — the default is !import.meta.env.PROD — and filtered out of the built site. Each draft also raises a lint warning in the error panel.

Tags: only first 2 shown

tags: [a, b, c, d]
  card → a  b

The card renders tags.slice(0, 2) — extra tags exist in frontmatter but the preview card only shows the first two. All tags are still available on the post page.

A NN_ prefix is meaningless

01_2024-01-15-x.md

Blogs sort by date, never by an order prefix. A leading 01_ is not stripped and not interpreted — it just becomes part of the slug/name. Use the date, not a number, to order posts.

Rules

RuleStatusDetail
title in frontmatter required The only required field; falls back to "Untitled" if missing.
YYYY-MM-DD- filename prefix recommended Not enforced, but it is how a post gets a date without frontmatter. Without a date (here or in frontmatter) the post sorts to the oldest end and shows no date.
Frontmatter date optional Overrides the filename date when present. Drives newest-first ordering and the card date.
Flat layout (no nesting) recommended Blog is a flat folder of files; there is no sidebar tree and no folder grouping.
NN_ order prefix n/a The docs prefix grammar does not apply — ordering is by date only. A number is not stripped.
draft: true optional Dev-only. Shown while developing, filtered out of the production build; raises a lint warning.
description optional Shown on the card; its absence raises a "missing-description" lint warning (SEO hint).
author, tags, image optional All shown on the card when present; card renders at most the first 2 tags.
