# Dump: future features & ideas

The tracker's **issue dump** for unhomed feature ideas — thoughts that don't yet pass
the creation litmus test (component + first subtask nameable in one breath) and have
no existing issue to attach to. Nothing here is scheduled; the dump exists so ideas
aren't lost without paying full-issue ceremony for each one.

## The dump contract

- **Each entry is a subtask** (currently grouped by category — content, navigation,
  visuals, social, performance, analytics, dev tooling).
- **Capture** new entries via `/docs-quick-idea-note` (or by hand: next `NN_`
  subtask, a couple of context-setting sentences — never a bare fragment).
- **Graduation = promote and delete.** When an entry can name its component and
  first subtask, it becomes a real issue and the entry is **deleted from here** —
  not ticked off. A dump's value is its current contents; git history keeps
  provenance.
- Only for the **unhomed** — a thought that obviously belongs to an existing issue
  goes to that issue's `subtasks/` directly.

Full rules: `2026-07-01-issue-anatomy-restructure/notes/02_operating-rules.md` (and
the `agent-ks-issues` skill).
