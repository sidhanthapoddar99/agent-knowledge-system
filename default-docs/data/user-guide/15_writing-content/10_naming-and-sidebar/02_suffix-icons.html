Trailing type glyphs

A single glyph can trail the label to mark a non-markdown page. Markdown is the default and is always unmarked — a glyph appears only for the exceptions. This vocabulary is shared by both the docs sidebar and the issue sub-doc tree (docs call the field fileType, issues call it docType — the marked keys are identical). Artifact both

A self-contained .html artifact page (a report, dashboard, or explainer like this one). Tooltip label: Artifact.

Two, and only two, marked types. The glyph map contains exactly diagram and artifact. Any other file type — including markdown — resolves to no glyph. Growing the vocabulary means adding a keyed entry to file-type-icons.ts; there is no per-page override.
