# Notes

**Scope:** finalized output + durable references — the decided approach you build on, plus
research/links. The *product*.

Notes are defined by **contrast with Brainstorm**: brainstorm is *what we're figuring
out*, notes are *what we know*. Content arrives either by **graduating** out of a resolved
brainstorm (the conclusion distilled, the trail left behind) or fully formed (a reference,
a how-to, an external link dump). Once here it should be **stable** — a note that keeps
changing is a brainstorm wearing the wrong hat.

Shape: the standard free-form 2-level tree, plain `NN_<slug>.md` — **no kind words, no
machinery**. The numbering is a *curated reading order* (the author's intended sequence),
not a timeline, which is why notes keep plain ascending order and need nothing else.

## Decided

- **"Good as it is"** — no loader or UI changes; the generic subdoc treatment covers it.
- No kind vocabulary — kinds mark *modes of deliberation*; notes have one mode (settled).
- Fed by brainstorm graduation; the brainstorm keeps the *why*, the note keeps the *what*.
