Docs — File Tree → Sidebar

A docs section's folder tree is its navigation. Every file needs an NN_ prefix (2–5 digits) — this one is mandatory: a docs file without it is a hard build error. The sidebar then throws the prefix away and shows the frontmatter title; the number survives only as sort order.

On disk → in the sidebar

A realistic section: root files interleave with folder sections, sorted together by the numeric value of their prefix. Every folder becomes a collapsible section header; every file becomes a row labelled by its frontmatter title, never its filename.

Read this pair carefully. The NN_ prefixes and the getting-started/ folder name are gone from the sidebar. Labels come from each file's title frontmatter; a folder becomes a section header named from its own settings.json label (or its title-cased slug). Ordering follows the numeric prefix value — so 05_ (5) < 010_ (10) < 110_ (110), and the root file 01_overview.md lands above them all. Source: Sidebar.astro, useSidebar.ts lines 223–237, 271–288.

Edge cases

Every rule and every way it can go wrong — a hard error, a silent warning, or a quiet sort.

Missing prefix → build fails

guides/setup.md ✗ no NN_ prefix

[DOCS ERROR] Files missing required XX_ position prefix: - guides/setup.md Docs files must be named with a position prefix (01-99). Examples: 01_getting-started.mdx 02_installation.mdx guides/01_configuration.mdx

Verbatim throw from data.ts lines 253–265. Docs is the only content type where a missing prefix is fatal — the site won't start.

Mixed widths sort by value

05_a.md → 5 010_b.md → 10 110_c.md → 110

Order is the parsed numeric value, never the string — so 05 < 010 < 110. Gap-space your prefixes (10, 20, 30) to insert later without renumbering. order-prefix.ts.

A folder is a section

05_getting-started/ ├─ settings.json ├─ 01_intro.md └─ 10_install.md

Every prefixed sub-folder renders as a collapsible section header (from its settings.json label, else the title-cased slug); its files nest one level in. useSidebar.ts nodeToSection().

sidebar_position overrides

50_appendix.md --- title: Appendix sidebar_position: 1 # wins

The prefix seeds sidebar_position (base-parser.ts 184–189); an explicit frontmatter value overrides it. So 50_appendix.md can jump to the top without renaming — and acts as the tiebreak when two prefixes collide.

Deeper than the render depth

10_a/ › 10_b/ › 10_c/ › 10_d/
The default docs sidebar draws about 3 levels (sidebar.depth default 3, data.ts 383). Pages nested deeper still build and are reachable by URL — they just don't appear as sidebar rows.

Docs have no hard nesting cap — you can nest as deep as you like and every page builds and routes. sidebar.depth only limits how deep the sidebar draws. (The 5-level hard cap, MAX_SUBFOLDER_DEPTH, is the separate issue-tracker rule — it does not apply to docs.)

Non-markdown page + glyph

05_architecture.excalidraw 08_dashboard.html
System Architecture
Dashboard

.mmd/.dot/.excalidraw and .html pages render first-class (still NN_-prefixed) and carry a trailing type glyph — the only mark a docs row ever gets. Enabled by default; opt out with allow_diagram_pages / allow_artifact_pages: false. Unlike markdown, a missing prefix here is a warning + skip, not a build error. file-type-icons.ts, diagram-pages.ts 20–23, artifact-pages.ts 32–37.

Label = title, not filename

05_site-yaml.md --- title: Site Configuration
Site Configuration

The row reads Site Configuration — never site-yaml. title frontmatter is required per file (docs.ts schema); a sidebar_label can shorten it. Missing title throws (base-parser.ts 285–290).

One digit / six digits ≠ prefix

1_intro.md # NOT a prefix 123456_x.md # NOT a prefix

The grammar is strictly 2–5 digits then _. A single digit or 6+ digits isn't parsed as a prefix — so 1_intro.md counts as no prefix and triggers the hard build error above. order-prefix.ts 24–25, 41.

Rules at a glance

RuleStatusDetail
NN_ prefix on every docs file required Mandatory — the loader throws a ParserError at build if any docs file lacks it. Docs is the only content type that enforces this.
NN_ prefix on sub-folders convention Sets section order by numeric value. Unprefixed folders default to position 999 and sort last rather than erroring.
settings.json per folder required Declares the section label, collapse behaviour, sidebar depth, outline, pagination. Missing → framework defaults (validator flags it).
title frontmatter per file required Becomes the sidebar label (the prefix and slug are discarded). Missing title throws.
sidebar_position / sidebar_label optional Override the prefix-derived order and shorten the label. Absent → order comes from the prefix value.
Prefix width 2–5 digits Sort by numeric value, so mixed widths coexist (05 < 010 < 110). One digit or 6+ digits is not a prefix.
Folder nesting depth unlimited Docs have no hard cap — nest freely, all pages build and route. The sidebar only draws to sidebar.depth (default 3); deeper pages exist but aren't shown as rows. (The 5-level MAX_SUBFOLDER_DEPTH cap is the issue tracker's, not docs'.)
Non-markdown pages (.mmd/.dot/.excalidraw/.html) opt-out First-class when NN_-prefixed; on by default, disable with allow_diagram_pages/allow_artifact_pages: false. Carry a trailing type glyph; missing prefix = warning, not error.
